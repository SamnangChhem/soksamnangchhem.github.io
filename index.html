Core Strategy

Effective UI design combine business objectives, product goals, and user needs through proper design stretegy.

Know the User

Start with user needs. Treat second-hand information about users with caution. Focus on user outcomes and design with data.

Clarity

Create structure and hierarchy. Give users responses to the actions they take. Make decisions easy by avoiding paradox of choice.

Consistency

Avoid uniformity when unsuitable but be ensure consistent. Re-use well-tested design patterns. Use ubiquitous language to create familiarity.

Effectiveness

Designs that works everywhere for everyone. Produce interfaces that prioritize inclusive design enhance user experience by foster a sense of belonging by leverage technology to help users.

My Process



Gain real insight into users and their needs. Insights gathered during the Empathize stage and is carried on to the next stages to help define the problem and understand how to deal with it.

1. Emphasize

Define the problem, identifying the users’ needs through problem statement, a hypothesis statement, and a value proposition. Explore how psychology and human factors influence design.

2. Define

Brainstorm ideas for design solutions with knowledge gained from prior steps. Conduct a competitive audit and complete design activities, using How Might We and Crazy Eights methods.

3. Ideate

Turning ideas into concrete examples. Create wireframes and prototypes.Gives something tangible to test on real and potential users, which is crucial in making sure that our designs are usable.

4. Prototype

Improve upon the original design. Eliminate problems or user difficulties that were unforeseen in the design phase before getting started on the implementation phase.

5. Test
